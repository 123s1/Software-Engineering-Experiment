# 实验3：教室预订系统持久化设计（数据库建模）

## 1. 文档说明

本文件依据实验1的需求分析和实验2的类图设计，对"教室预订系统"进行实验3的数据库建模设计。

本次数据库设计的核心目标：

- 将实验2中的核心实体类映射为数据库表
- 通过中间表拆分多对多关系，保持数据规范化
- 明确主键、外键、约束和索引
- 时间粒度精确到"课时"
- 能够支撑系统的所有核心查询与业务操作

## 2. 设计依据

### 2.1 来自实验1的需求依据

- 管理员可增加/删除用户、增加/删除教室、设置可用时间段、浏览预订情况
- 师生可登录、修改密码、浏览空闲教室、预订教室
- 时间粒度精确到"课时"
- 需支持最多 10000 人同时在线

### 2.2 来自实验2的类图依据

实验2中确定的核心实体类直接对应为数据库表：

| 实验2类名 | 对应数据库表 | 说明 |
| --- | --- | --- |
| `User` | `t_user` | 用户主表（父表） |
| `Administrator` | `t_administrator` | 管理员子表 |
| `TeacherStudentUser` | `t_teacher_student` | 师生用户子表 |
| `Classroom` | `t_classroom` | 教室资源表 |
| `ClassroomTimeSlot` | `t_classroom_time_slot` | 教室可用时间段表 |
| `Reservation` | `t_reservation` | 预订记录表（拆分师生-教室 N:M） |
| `ClassroomManagementRecord` | `t_classroom_mgmt_record` | 教室管理记录表（拆分管理员-教室 N:M） |
| `UserManagementRecord` | `t_user_mgmt_record` | 用户管理记录表（拆分管理员-师生 N:M） |

### 2.3 继承关系的数据库映射策略

实验2中 `Administrator` 和 `TeacherStudentUser` 继承自 `User`。

**本次采用多表继承方案**：

- 建一个用户主表 `t_user`，存放所有用户的公共属性
- 建管理员子表 `t_administrator`，通过外键指向 `t_user`
- 建师生子表 `t_teacher_student`，通过外键指向 `t_user`，并存放师生独有属性

**选择多表继承的原因**：

1. 管理员和师生的**功能完全不同**：管理员负责管理用户/教室，师生负责查询/预订
2. 分表后，管理员和师生各自的关联关系更加清晰（管理记录指向管理员表，预订记录指向师生表）
3. 与实验2类图中的继承关系完全对应
4. 在 CDM 中可以直接用继承关系表达，生成 PDM 时自动产生外键

## 3. 多对多关系拆分设计

本系统中存在三组多对多（N:M）关系，每组都通过引入中间表拆分为两个一对多（1:N）关系：

### 3.1 师生与教室的预订关系（N:M）

**问题**：多名师生可以预订多间教室，多个教室也可以被多名师生预订。

**解决**：通过 `t_reservation`（预订记录表）拆分。

```
t_teacher_student (1) ──── (0..*) t_reservation (0..*) ──── (1) t_classroom
```

- `t_teacher_student` (1) → (0..*) `t_reservation`：一个师生用户可以发起多条预订
- `t_reservation` (0..*) → (1) `t_classroom`：多条预订可以指向同一教室
- `t_reservation` (0..*) → (1) `t_classroom_time_slot`：每条预订占用一个具体时间段

### 3.2 管理员与教室的管理关系（N:M）

**问题**：多个管理员可以管理（上架、下架、调整时段）多个教室。

**解决**：通过 `t_classroom_mgmt_record`（教室管理记录表）拆分。

```
t_administrator (1) ──── (0..*) t_classroom_mgmt_record (0..*) ──── (1) t_classroom
```

- `t_administrator` (1) → (0..*) `t_classroom_mgmt_record`：一个管理员可以产生多条教室管理操作记录
- `t_classroom_mgmt_record` (0..*) → (1) `t_classroom`：一个教室可以有多条被管理的历史记录

### 3.3 管理员与师生用户的管理关系（N:M）

**问题**：多个管理员可以管理（启用、禁用）多名师生用户。

**解决**：通过 `t_user_mgmt_record`（用户管理记录表）拆分。

```
t_administrator (1) ──── (0..*) t_user_mgmt_record (0..*) ──── (1) t_teacher_student
```

- `t_administrator` (1) → (0..*) `t_user_mgmt_record`：一个管理员可以产生多条用户管理操作记录
- `t_user_mgmt_record` (0..*) → (1) `t_teacher_student`：一个师生可以有多条被管理的历史记录

## 4. 数据表详细设计

### 4.1 用户主表（t_user）

| 字段名 | 数据类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| user_id | VARCHAR(32) | PRIMARY KEY | 用户唯一编号 |
| account | VARCHAR(50) | NOT NULL, UNIQUE | 登录账号，必须唯一 |
| user_name | VARCHAR(50) | NOT NULL | 用户姓名 |
| password | VARCHAR(128) | NOT NULL | 密码（建议加密存储） |
| contact_info | VARCHAR(100) | NULL | 联系方式 |
| user_status | VARCHAR(20) | NOT NULL, DEFAULT 'ACTIVE' | 用户状态：ACTIVE（启用）/ DISABLED（停用） |

**主键**：`user_id`

**唯一约束**：`account`

### 4.2 管理员子表（t_administrator）

| 字段名 | 数据类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| admin_id | VARCHAR(32) | PRIMARY KEY | 管理员唯一编号 |
| user_id | VARCHAR(32) | NOT NULL, FOREIGN KEY → t_user(user_id) | 关联的用户主表编号（继承关系） |

**主键**：`admin_id`

**外键**：`user_id` → `t_user(user_id)`

### 4.3 师生用户子表（t_teacher_student）

| 字段名 | 数据类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| ts_id | VARCHAR(32) | PRIMARY KEY | 师生用户唯一编号 |
| user_id | VARCHAR(32) | NOT NULL, FOREIGN KEY → t_user(user_id) | 关联的用户主表编号（继承关系） |
| identity_type | VARCHAR(20) | NOT NULL | 身份类型：TEACHER（教师）/ STUDENT（学生） |

**主键**：`ts_id`

**外键**：`user_id` → `t_user(user_id)`

### 4.4 教室表（t_classroom）

| 字段名 | 数据类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| classroom_id | VARCHAR(32) | PRIMARY KEY | 教室唯一编号 |
| classroom_name | VARCHAR(50) | NOT NULL | 教室名称 |
| location | VARCHAR(100) | NOT NULL | 教室位置 |
| capacity | INT | NOT NULL | 教室容量（座位数） |
| equipment_info | VARCHAR(200) | NULL | 设备信息（如多媒体设备情况） |
| classroom_status | VARCHAR(20) | NOT NULL, DEFAULT 'AVAILABLE' | 教室状态：AVAILABLE（可预订）/ DISABLED（停用） |

**主键**：`classroom_id`

### 4.5 教室可用时间段表（t_classroom_time_slot）

| 字段名 | 数据类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| slot_id | VARCHAR(32) | PRIMARY KEY | 时间段唯一编号 |
| classroom_id | VARCHAR(32) | NOT NULL, FOREIGN KEY → t_classroom(classroom_id) | 所属教室编号 |
| use_date | DATE | NOT NULL | 使用日期 |
| start_section | INT | NOT NULL | 开始课时（如第1节） |
| end_section | INT | NOT NULL | 结束课时（如第4节） |
| slot_status | VARCHAR(20) | NOT NULL, DEFAULT 'FREE' | 时间段状态：FREE（空闲）/ RESERVED（已预订）/ UNAVAILABLE（不可用） |

**主键**：`slot_id`

**外键**：`classroom_id` → `t_classroom(classroom_id)`

**业务约束**：`start_section <= end_section`；同一教室同一日期的时间段不应重叠

### 4.6 预订记录表（t_reservation）——拆分师生与教室 N:M 关系

| 字段名 | 数据类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| reservation_id | VARCHAR(32) | PRIMARY KEY | 预订记录唯一编号 |
| ts_id | VARCHAR(32) | NOT NULL, FOREIGN KEY → t_teacher_student(ts_id) | 申请人（师生用户编号） |
| classroom_id | VARCHAR(32) | NOT NULL, FOREIGN KEY → t_classroom(classroom_id) | 被预订教室编号 |
| slot_id | VARCHAR(32) | NOT NULL, FOREIGN KEY → t_classroom_time_slot(slot_id) | 占用的时间段编号 |
| purpose | VARCHAR(200) | NULL | 预订用途 |
| reservation_status | VARCHAR(20) | NOT NULL, DEFAULT 'CREATED' | 预订状态：CREATED（已创建）/ CANCELLED（已取消）/ FINISHED（已完成） |
| create_time | DATETIME | NOT NULL | 预订创建时间 |

**主键**：`reservation_id`

**外键**：
- `ts_id` → `t_teacher_student(ts_id)`
- `classroom_id` → `t_classroom(classroom_id)`
- `slot_id` → `t_classroom_time_slot(slot_id)`

**本表的核心作用**：将师生用户与教室之间的多对多预订关系，拆分为"师生 (1) → (0..*) 预订记录"和"预订记录 (0..*) → (1) 教室"两个一对多关系。

### 4.7 教室管理记录表（t_classroom_mgmt_record）——拆分管理员与教室 N:M 关系

| 字段名 | 数据类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| record_id | VARCHAR(32) | PRIMARY KEY | 管理记录唯一编号 |
| admin_id | VARCHAR(32) | NOT NULL, FOREIGN KEY → t_administrator(admin_id) | 操作人（管理员编号） |
| classroom_id | VARCHAR(32) | NOT NULL, FOREIGN KEY → t_classroom(classroom_id) | 被操作教室编号 |
| operation_type | VARCHAR(20) | NOT NULL | 操作类型：ADD（增加教室）/ DELETE（删除教室）/ SET_TIMESLOT（设置时间段） |
| operation_time | DATETIME | NOT NULL | 操作时间 |
| remarks | VARCHAR(500) | NULL | 备注信息 |

**主键**：`record_id`

**外键**：
- `admin_id` → `t_administrator(admin_id)`
- `classroom_id` → `t_classroom(classroom_id)`

**本表的核心作用**：将管理员与教室之间的多对多管理关系，拆分为"管理员 (1) → (0..*) 教室管理记录"和"教室管理记录 (0..*) → (1) 教室"两个一对多关系。同时充当操作审计日志。

### 4.8 用户管理记录表（t_user_mgmt_record）——拆分管理员与师生 N:M 关系

| 字段名 | 数据类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| record_id | VARCHAR(32) | PRIMARY KEY | 管理记录唯一编号 |
| admin_id | VARCHAR(32) | NOT NULL, FOREIGN KEY → t_administrator(admin_id) | 操作人（管理员编号） |
| target_ts_id | VARCHAR(32) | NOT NULL, FOREIGN KEY → t_teacher_student(ts_id) | 被操作的目标师生用户编号 |
| operation_type | VARCHAR(20) | NOT NULL | 操作类型：ENABLE（启用）/ DISABLE（禁用） |
| operation_time | DATETIME | NOT NULL | 操作时间 |
| remarks | VARCHAR(500) | NULL | 备注信息 |

**主键**：`record_id`

**外键**：
- `admin_id` → `t_administrator(admin_id)`
- `target_ts_id` → `t_teacher_student(ts_id)`

**本表的核心作用**：将管理员与师生用户之间的多对多管理关系，拆分为"管理员 (1) → (0..*) 用户管理记录"和"用户管理记录 (0..*) → (1) 师生"两个一对多关系。同时充当操作审计日志。

## 5. 主外键关系汇总

| 序号 | 外键所在表 | 外键字段 | 引用表 | 引用字段 | 关系说明 |
| --- | --- | --- | --- | --- | --- |
| 1 | t_administrator | user_id | t_user | user_id | 管理员继承自用户 |
| 2 | t_teacher_student | user_id | t_user | user_id | 师生继承自用户 |
| 3 | t_classroom_time_slot | classroom_id | t_classroom | classroom_id | 时间段从属于教室（1:N） |
| 4 | t_reservation | ts_id | t_teacher_student | ts_id | 预订由师生发起（1:N） |
| 5 | t_reservation | classroom_id | t_classroom | classroom_id | 预订指向教室（1:N） |
| 6 | t_reservation | slot_id | t_classroom_time_slot | slot_id | 预订占用时间段（1:N） |
| 7 | t_classroom_mgmt_record | admin_id | t_administrator | admin_id | 教室管理由管理员发起（1:N） |
| 8 | t_classroom_mgmt_record | classroom_id | t_classroom | classroom_id | 教室管理针对教室（1:N） |
| 9 | t_user_mgmt_record | admin_id | t_administrator | admin_id | 用户管理由管理员发起（1:N） |
| 10 | t_user_mgmt_record | target_ts_id | t_teacher_student | ts_id | 用户管理针对师生（1:N） |

## 6. 三组多对多关系拆分总结

| 序号 | 原始多对多关系 | 中间表 | 拆分后的两个一对多关系 |
| --- | --- | --- | --- |
| 1 | 师生 ↔ 教室（预订） | t_reservation | t_teacher_student (1)→(0..*) t_reservation；t_reservation (0..*)→(1) t_classroom |
| 2 | 管理员 ↔ 教室（管理） | t_classroom_mgmt_record | t_administrator (1)→(0..*) t_classroom_mgmt_record；t_classroom_mgmt_record (0..*)→(1) t_classroom |
| 3 | 管理员 ↔ 师生（管理） | t_user_mgmt_record | t_administrator (1)→(0..*) t_user_mgmt_record；t_user_mgmt_record (0..*)→(1) t_teacher_student |

## 8. 数据库设计与业务需求的对应关系

### 8.1 功能覆盖验证

| 需求功能 | 涉及的数据表 | 说明 |
| --- | --- | --- |
| 登录系统 | t_user | 根据 account + password 校验 |
| 修改密码 | t_user | 更新 password 字段 |
| 增加用户 | t_user, t_teacher_student, t_user_mgmt_record | 插入用户和师生记录，生成管理日志 |
| 删除/停用用户 | t_user, t_user_mgmt_record, t_reservation | 更新用户状态，处理关联预订，生成管理日志 |
| 增加教室 | t_classroom, t_classroom_mgmt_record | 插入教室记录，生成管理日志 |
| 删除/停用教室 | t_classroom, t_classroom_mgmt_record, t_reservation | 更新教室状态，处理关联预订，生成管理日志 |
| 设置教室可用时间段 | t_classroom_time_slot, t_classroom_mgmt_record | 插入时间段记录，生成管理日志 |
| 浏览空闲教室 | t_classroom, t_classroom_time_slot | 按日期和课时查询 slot_status='FREE' 且 classroom_status='AVAILABLE' |
| 预订教室 | t_reservation, t_classroom_time_slot | 插入预订记录，更新时间段状态为 RESERVED |
| 浏览预订情况 | t_reservation, t_teacher_student, t_user, t_classroom, t_classroom_time_slot | 多表联查，支持按时间段筛选 |

### 8.2 时间精确到"课时"的体现

- `t_classroom_time_slot` 表使用 `start_section`（开始课时）和 `end_section`（结束课时）两个整数字段
- 课时以整数编号，如第1节、第2节……第10节
- 冲突校验通过比较同一教室、同一日期下的课时范围是否重叠来实现

### 8.3 并发性能考虑

- 对高频查询字段建立索引
- 预订时应使用数据库事务保证原子性
- 可通过乐观锁或悲观锁防止同一时段被重复预订

## 9. 与实验2的衔接

| 实验2的设计 | 实验3的数据库映射 |
| --- | --- |
| `User` 抽象父类 | `t_user` 用户主表 |
| `Administrator` 继承 `User` | `t_administrator` 子表，外键指向 `t_user` |
| `TeacherStudentUser` 继承 `User` | `t_teacher_student` 子表，外键指向 `t_user` |
| `Classroom` 组合 `ClassroomTimeSlot` | `t_classroom` + `t_classroom_time_slot`（外键关联） |
| `Reservation` 关联 User/Classroom/TimeSlot | `t_reservation` 含三个外键 |
| `ClassroomManagementRecord` 拆分管理员-教室 N:M | `t_classroom_mgmt_record` 含两个外键 |
| `UserManagementRecord` 拆分管理员-师生 N:M | `t_user_mgmt_record` 含两个外键 |
| 服务类（AuthenticationService 等） | 不映射为表（属于应用层逻辑） |
